# Coleman Parks — Claude Code Project Context

## What This Repo Is

Client website builds and a brand playbook. Every subfolder is a live client site deployed via GitHub Pages. Coleman builds marketing and brand websites for clients — HTML, CSS, JavaScript only. No frameworks. No build tools.

## Register: Brand

All work here is **brand register** — marketing sites, landing pages, client-facing brand work. The goal is distinctiveness. A polished, non-generic site that reflects the client's brand positioning, not a template.

## Workflow

1. Generate HTML from Claude.ai prompt
2. `./new-client.sh "Name" "slug"` — scaffold the folder
3. Paste HTML into `slug/index.html`
4. `./check.sh slug/index.html` — run design quality check
5. Fix flagged issues
6. `git add . && git commit -m "build: slug" && git push`

# Design Laws (Non-Negotiable)

These rules apply to every HTML file generated or edited in this repo. They are sourced from Impeccable, Vercel's web-design-guidelines, and Anthropic's frontend-design skill.

---

## Color

- Use OKLCH color space. Reduce chroma near 0 or 100 lightness.
- **Never use `#000` or `#fff`** — always tint neutrals toward brand hue (chroma 0.005–0.01 minimum).
- Never use gray text on colored backgrounds — always check contrast.
- Never use pure gray — always tint toward brand.
- Color commitment levels for brand sites: default to **Committed** (one saturated color covering 30–60% of the surface).
- No purple gradients. No blue-to-purple gradients. No rainbow/holographic gradients on text.
- `background-clip: text` with gradients is banned. Use solid color, weight, or size for emphasis.

## Typography

- Body line length: **65–75 characters maximum**. Never let paragraphs span full width.
- Hierarchy via scale + weight contrast. Minimum 1.25 ratio between steps. No flat type scales.
- Never use Arial, system-ui, or generic sans-serif as the primary font — use Google Fonts.
- H1 must be present and unique on every page.
- Never use `line-height` below 1.4 for body text.
- Font sizes: body minimum 16px, captions minimum 13px.
- Never skip heading levels (H1 → H3 with no H2).

## Layout and Spacing

- Vary spacing for rhythm — monotonous padding everywhere is an anti-pattern.
- **Cards are lazy defaults.** Use only when genuinely the best affordance.
- **Nested cards are always wrong.**
- Don't wrap everything in containers. Most elements don't need one.
- Touch targets minimum 44px × 44px.
- No horizontal scroll on mobile at any viewport.
- Container max-width: 1160px, centered with `margin: 0 auto`.

## Motion and Animation

- **Never animate CSS layout properties** (width, height, margin, padding, top, left).
- Use exponential ease-out curves: `cubic-bezier(0.16, 1, 0.3, 1)` or similar ease-out-quart/quint.
- **No bounce or elastic easing.** It reads as dated immediately.
- Scroll reveal: opacity + translateY(20px), threshold 0.08, rootMargin -40px.
- Never autoplay video with sound.
- Respect `prefers-reduced-motion` — wrap animations in `@media (prefers-reduced-motion: no-preference)`.

## Absolute Bans — Rewrite Rather Than Use

- **Side-stripe borders:** Colored `border-left` or `border-right` > 1px on cards, callouts, alerts. Rewrite with full borders, background tints, or icons.
- **Glassmorphism as default:** Blurs and glass cards used decoratively. Reserve for rare, purposeful moments only.
- **Hero-metric template:** Big number + small label + supporting stats + gradient accent. Overused SaaS pattern.
- **Identical card grids:** Same-sized cards with icon + heading + text repeated 3 times. Monotonous.
- **Modal-first design:** Exhaust inline and progressive disclosure alternatives before reaching for a modal.
- **Dark glows:** Glowing shadows on dark backgrounds (box-shadow with large blur + color).
- **Bounce/elastic easing** on any animation.

## AI Slop Test (Run Before Every Push)

Ask at two altitudes before shipping:

1. **Domain reflex test:** Could the category alone predict this aesthetic? ("fitness brand → orange and black", "law firm → navy and gold") — if yes, rework the color and scene strategy.
2. **Anti-reference test:** Could someone immediately identify this as AI-generated by the pattern of choices? If yes, make more committed decisions.

## Accessibility (WCAG AA — Non-Negotiable)

- **Skip link** must be the first element in `<body>`: `<a href="#main" class="skip-link">Skip to main content</a>`
- `<main id="main">` must be present.
- All `<img>` must have `alt` text. Decorative images use `alt=""`.
- All form inputs must have associated `<label>` elements (use `for`/`id` pairing).
- All interactive elements must have visible focus states.
- Color contrast: minimum 4.5:1 for normal text, 3:1 for large text.
- Navigation: `<nav role="navigation" aria-label="Main navigation">`.
- Hamburger button: `aria-label`, `aria-expanded`, `aria-controls`.
- Buttons must have descriptive labels — no "click here" or "learn more" alone.
- Never remove `outline` without replacing it with a visible focus style.

---

## Performance

- Lazy load all images below the fold: `loading="lazy"`.
- Google Fonts: always use `display=swap` in the font URL.
- Inline critical CSS in `<style>` — no render-blocking external CSS for above-fold content.
- Compress and size images correctly — no 2MB images for a 400px thumbnail.
- No unused JavaScript libraries.

---

## SEO Minimum

Every page must have:
- Unique `<title>` (under 60 characters)
- Unique `<meta name="description">` (under 160 characters)
- One `<h1>` that matches page intent
- `<meta property="og:title">` and `<meta property="og:description">`

---

## HTML Quality

- Semantic HTML5: `<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`, `<article>`.
- No `<div>` where a semantic element exists.
- No inline styles (except for dynamic values set by JavaScript).
- `lang="en"` on the `<html>` element.
- Viewport meta must be present.
- Favicon must be linked.

---

## The Non-Generic Mandate

Coleman Parks clients are paying for a brand — not a template. Before every build:

1. Read the brand brief carefully.
2. Make a specific, committed color choice that fits this client — not the industry default.
3. Choose typography that reflects the brand personality — not Inter.
4. Write real copy based on the brief — no placeholder text, ever.
5. The site must not look like it came from a template generator.
