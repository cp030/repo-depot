01

Tools Overview

Everything you need to build websites and brands from scratch. Each tool has a specific role — understand the system before diving in.

Code Editor

VS Code

Where you write and edit code. Free. Download from code.visualstudio.com. Has a built-in terminal so you can do everything in one window.

AI Code Assistant

Claude Code

AI that works inside your codebase from the terminal. Edits files, fixes bugs, builds features, runs commands. Installed via terminal on Mac.

Version Control

Git + GitHub

Git tracks changes to your code. GitHub stores it online. Together they let you deploy websites for free using GitHub Pages.

Visual Design

Claude Design

Builds visual brand assets, prototypes, logos, and mockups. Lives inside claude.ai — click the palette icon in the left sidebar.

File Management

Claude Cowork

Desktop tool for automating file organization, asset resizing, and task management. Handles the non-code side of brand delivery.

Planning

Claude.ai (this chat)

Planning, writing, prompting, reviewing. Generate website HTML here then paste into VS Code or the Brand Studio artifact to preview.

The Workflow in One Line

Claude.ai plans and generates → Claude Code in VS Code builds and edits → Git pushes to GitHub → GitHub Pages makes it live → Claude Design handles visuals

Mac Setup — From Zero

Do these in order. Each step depends on the one before it.

Step 1 — Install Git

Open Terminal (Command + Space, type Terminal, hit Enter) and run:

Check if Git is already installed
git --version

If you get a version number, Git is already installed. If you get an error, type git and macOS will prompt you to install it automatically via Xcode Command Line Tools. Click Install and wait.

Step 2 — Install VS Code

Download from code.visualstudio.com. Drag it to your Applications folder. Open it.

Then register the code command so you can open folders from Terminal:

1

Open VS Code

Launch it from your Applications folder.

2

Open Command Palette

Press Command + Shift + P

3

Type and select

Shell Command: Install 'code' command in PATH then hit Enter

4

Test it

Open Terminal and type code . — VS Code should open your current folder

Step 3 — Install Claude Code

In Terminal, paste and run:

curl -fsSL https://claude.ai/install.sh | sh

Then verify it installed correctly:

claude doctor

You should see green checkmarks. Then log in:

claude

It will prompt you to log in with your Claude account in the browser on first launch. After that, credentials are saved.

Step 4 — Install GitHub CLI (for easy authentication)

First install Homebrew (the Mac package manager):

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

It will ask for your Mac password — type it and hit Enter. Nothing shows while you type, that is normal. Then:

brew install gh gh auth login

Follow the prompts — select GitHub.com, HTTPS, and authenticate in browser. Your Mac password goes in the browser, not Terminal.

Common Confusion

Git is not GitHub. GitHub Pages is not GitHub. VS Code and Terminal are not Git. They are all separate things that work together. You never "open Git" — you run git commands inside Terminal or the VS Code terminal.

Website Building Workflow

The full process from blank folder to live website. Follow these stages in order.

Stage 1 — Plan the site in Claude.ai

Before writing a line of code, build the brief. Paste your client intake form responses here and ask:

"Generate a complete website for this brand. Return only raw HTML starting with <!DOCTYPE html>. Include: hero, about, services (3 items), why us, contact form with EmailJS, footer. Mobile-first responsive at 480px, 768px, 1024px. Use these exact colors and fonts: [paste brand brief]."

Claude generates the full HTML here in the chat. Copy it.

Stage 2 — Preview in Brand Studio artifact

Paste the HTML into the Brand Studio artifact (BrandStudio.jsx in your Claude Project). Hit Load Preview. Review every section. Edit in the HTML tab if needed.

Stage 3 — Push to Claude Code
1

Run the new client setup script

./new-client.sh "Client Name" "client-short-name"

This creates the full folder structure automatically

2

Paste the HTML into the website folder

Replace the index.html in Clients/client-short-name/06_Website/ with your exported HTML

3

Launch Claude Code and split the file

cd Clients/client-short-name/06_Website claude

Then tell Claude Code: "Split this single-file website into index.html, css/style.css, css/responsive.css, js/main.js, and js/form.js. Keep all functionality identical."

4

Wire up the contact form

Tell Claude Code: "Wire up the contact form using EmailJS with Public Key [key], Service ID [id], Template ID [id]. Validate all fields before submitting. Show sending state, success state, and error state."

5

Run the QA pass

Tell Claude Code: "Review every page for mobile responsiveness, accessibility (WCAG AA), SEO (title tags, meta descriptions, alt text), performance (lazy images, minified CSS/JS), and cross-browser Safari compatibility. Fix everything."

HTML Structure Every Site Needs

These are the non-negotiable elements that every professional website requires. Miss any of these and the site is incomplete.

Head Section Checklist
<!-- Every page needs all of these --> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Unique Page Title — Site Name</title> <meta name="description" content="Under 160 characters"> <!-- Open Graph for social sharing --> <meta property="og:title" content="…"> <meta property="og:description" content="…"> <meta property="og:type" content="website"> <meta property="og:url" content="https://yoursite.com"> <!-- Favicon --> <link rel="icon" type="image/png" sizes="32x32" href="images/favicon-32x32.png"> <!-- Google Fonts --> <link rel="preconnect" href="https://fonts.googleapis.com"> <link href="https://fonts.googleapis.com/css2?family=…" rel="stylesheet">
Accessibility Non-Negotiables
<!-- Skip link (first thing in body) --> <a href="#main" class="skip-link">Skip to main content</a> <!-- Nav needs role and aria-label --> <nav role="navigation" aria-label="Main navigation"> <!-- Hamburger button needs aria attributes --> <button aria-label="Toggle navigation menu" aria-expanded="false" aria-controls="navMobile"> <!-- Main content needs id="main" --> <main id="main"> <!-- All images need alt text --> <img src="logo.png" alt="Company Name" width="200" height="60" loading="lazy"> <!-- Form inputs need labels --> <label for="email">Email Address</label> <input type="email" id="email" name="email" required>

Animations and Motion

These patterns make sites feel modern and alive. Used in all three SPN options.

Scroll Reveal (Used on Every Section)
/* CSS */ .reveal { opacity: 0; transform: translateY(20px); transition: opacity 0.6s cubic-bezier(0.16, 1, 0.3, 1), transform 0.6s cubic-bezier(0.16, 1, 0.3, 1); } .reveal.visible { opacity: 1; transform: none; } /* JS — IntersectionObserver */ const observer = new IntersectionObserver(entries => { entries.forEach(e => { if (e.isIntersecting) { e.target.classList.add('visible'); observer.unobserve(e.target); } }); }, { threshold: 0.08, rootMargin: '0px 0px -40px 0px' }); document.querySelectorAll('.reveal').forEach((el, i) => { el.style.transitionDelay = (i % 5) * 0.08 + 's'; observer.observe(el); });
Counter Animation (Stats That Count Up)
/* HTML */ <span class="count" data-target="28">0</span> /* JS */ function animateCounter(el, target, duration = 1600) { let start = null; const step = timestamp => { if (!start) start = timestamp; const progress = Math.min((timestamp - start) / duration, 1); const eased = 1 - Math.pow(1 - progress, 3); el.textContent = Math.floor(eased * target); if (progress < 1) requestAnimationFrame(step); else el.textContent = target; }; requestAnimationFrame(step); } const countObs = new IntersectionObserver(entries => { entries.forEach(e => { if (e.isIntersecting) { animateCounter(e.target, +e.target.dataset.target); countObs.unobserve(e.target); } }); }, { threshold: 0.5 }); document.querySelectorAll('.count').forEach(el => countObs.observe(el));
Scrolling Ticker
/* HTML */ <div class="ticker"><div class="ticker-track" id="tick"></div></div> /* CSS */ .ticker { overflow: hidden; } .ticker-track { display: flex; white-space: nowrap; animation: ticker 28s linear infinite; } @keyframes ticker { 0% { transform: translateX(0); } 100% { transform: translateX(-50%); } } /* JS — duplicate items for seamless loop */ const items = ["Item 1", "Item 2", "Item 3"]; const track = document.getElementById('tick'); track.innerHTML = items.concat(items) .map(i => `<span class="tick-item">${i}</span>`) .join('');
SVG Animations (Waveforms, Radars, Scans)
<!-- Animated waveform path --> <path stroke="#00C8FF" stroke-width="1.5" fill="none"> <animate attributeName="d" dur="3s" repeatCount="indefinite" values="M20 60 Q60 20 100 60 Q140 100 180 60 Q220 20 260 60; M20 60 Q60 40 100 60 Q140 80 180 60 Q220 40 260 60; M20 60 Q60 20 100 60 Q140 100 180 60 Q220 20 260 60"/> </path> <!-- Animated scanning line --> <line stroke="rgba(0,200,255,0.5)" stroke-width="1.5"> <animate attributeName="x1" from="0" to="100%" dur="3s" repeatCount="indefinite"/> <animate attributeName="x2" from="0" to="100%" dur="3s" repeatCount="indefinite"/> </line> <!-- Spinning element --> <g style="transform-origin: 100px 100px; animation: spin 8s linear infinite"> <!-- content rotates --> </g> <style>@keyframes spin { to { transform: rotate(360deg); } }</style>
Nav That Changes on Scroll
const nav = document.getElementById('nav'); window.addEventListener('scroll', () => { nav.classList.toggle('scrolled', window.scrollY > 60); }, { passive: true }); /* CSS */ .nav { background: transparent; transition: all 0.3s ease; } .nav.scrolled { background: rgba(255, 255, 255, 0.95); backdrop-filter: blur(12px); border-bottom: 1px solid rgba(0,0,0,0.08); box-shadow: 0 1px 12px rgba(0,0,0,0.06); }

Deploying to GitHub Pages

GitHub Pages turns your repo into a free live website. Every push automatically updates the live site.

From Terminal (Mac)
# Make sure you're in your project folder cd your-project # Stage, commit, and push git add . git commit -m "initial website build" git push

Then on GitHub: Go to your repo → Settings → Pages → Under Branch select main → Save. Wait 60 seconds. Your site is live at https://USERNAME.github.io/REPO-NAME.

Project in a Subfolder

To host a client site at a subfolder (like Coleman does for SPN):

# Create subfolder and add index.html inside it mkdir spndt # Put your HTML file in there named index.html git add . git commit -m "add client website" git push

Live at: https://USERNAME.github.io/REPO-NAME/spndt

Brand Brief Generation

The brand brief is the source of truth for everything that follows. Generate it from the form responses using this prompt.

Prompt to Generate the Brief
"You are a senior brand strategist. I am going to paste client intake form responses below. Analyze every answer and produce a structured Brand Brief with these sections: 1. COMPANY OVERVIEW — name, what they do, who they serve, stage, location 2. BRAND POSITIONING — 2 to 3 sentence positioning statement 3. TARGET AUDIENCE — detailed profile of ideal customer 4. BRAND PERSONALITY — tone, voice, adjectives, how it should feel 5. VISUAL DIRECTION — colors, style, light vs dark, inspirations, what to avoid 6. COMPETITIVE LANDSCAPE — competitors and how this brand differentiates 7. MESSAGING PILLARS — 3 to 4 core messages to communicate consistently 8. TAGLINE OPTIONS — 3 concepts based on the brief 9. DELIVERABLES — checklist of what to build 10. OPEN QUESTIONS — anything unclear before proceeding Be specific, not generic. Use their actual words. No filler. [PASTE FORM RESPONSES HERE]"
Review the Brief Before Moving On

Check that the positioning statement reflects what they actually said. Confirm the visual direction matches their inspirations and avoid list. If there are open questions that would change the design direction, reach out to the client before starting visual work.

Visual Identity in Claude Design

Open Claude Design from the palette icon in claude.ai. Work through these in order and get approval at each step before moving forward.

Step 1 — Logo Concepts
"Build 3 logo concepts for this brand. Brand brief: [paste brief] For each concept: - Show the primary logo (horizontal or stacked) - Describe the design thinking - Explain the style (wordmark, lettermark, icon plus text, abstract) - Show why it fits the brand positioning Make each concept meaningfully different from the others."
Step 2 — Color Palette
"Using the approved logo direction and this brand brief, build a complete color system: - 1 primary color (for logo, CTAs, key moments) - 1 to 2 secondary colors (support the primary) - 1 neutral dark (text, backgrounds) - 1 neutral light (backgrounds, white space) - 1 accent (used sparingly) For each color: hex code, RGB, CMYK, and usage note. Show the palette applied to a sample layout. Confirm WCAG AA accessibility contrast compliance."
Step 3 — Typography System
"Select a typography system using Google Fonts only. - Heading font that matches brand personality - Body font that pairs well and reads well at small sizes - Full size scale: H1 through H4, body, caption, label - Line height and letter spacing for each - Show applied to a sample layout"
Step 4 — Brand Guidelines Document
"Generate a complete Brand Guidelines document with: 1. Brand Overview — mission, values, personality 2. Logo Usage — do and don't, sizes, clear space, variations 3. Color System — all colors with codes, usage rules 4. Typography — full type scale, pairing rules, examples 5. Imagery Style — photography direction 6. Voice and Tone — how the brand writes, on and off-brand examples 7. Applications — show brand on business card, website header, social post"

EmailJS Reference

Template Variables

Your EmailJS template must use these variable names exactly. They map to what the JavaScript sends.

VariableWhat It Contains
{{to_email}}Who receives the email — put this in the To Email field in template settings
{{from_name}}Sender name
{{from_email}}Sender email
{{reply_to}}Reply-to address
{{subject}}Email subject line
{{message}}Full message body — put this in the content area
DNS Fix for Microsoft 365 Email

If a client bought a domain and Microsoft 365 but email is not working: go to admin.microsoft.com → Settings → Domains → the domain → Continue Setup. Microsoft shows the exact MX, TXT, and CNAME records to add. Log into the domain registrar and add them exactly. Email flows within a few hours. The MX record is the critical one.
